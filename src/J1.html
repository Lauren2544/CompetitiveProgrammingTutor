Variables, Expressions, and Statements

Variables
- One of the most powerful features of a programming language is the ability to manipulate variables
- A variable is a name that refers to a value
- Variable names can consist of a-z characters, 0-9 numbers, and underscores and are usually mnemonic E.g. pizza, slices = 3, 5 (this will mean the variable with the name pizza equals 3 and slices equals 5)
- If you put a space in a variable name, or start with a number you will get a SyntaxError
- Python also has 35 keywords reserved and these cannot be variable names: and, del, from, None, True, as, elif, global, nonlocal, try, assert, else, if, not, while, break, except, import, or, with, class, False, in, pass, yield, continue, finally, is, raise, async, def, for, lambda, return, await

Values and Types
- In programming there are many different types of variables, these variables include text, numeric, sequence, mapping, set, Boolean, and binary types
- If you are not sure what type a value has, the id can tell you, E.g. print(type(“Hello World”))
- You can also set a variable to a specific type, E.g. x=input(int(5))
Text type:
--> string (you can identify strings because they are enclosed in quotations)
Numeric types:
--> integers
--> floats (decimal numbers)
--> complex numbers (have a real and imaginary part, which are each a floating-point number)
Sequence Types:
--> list (used to store data, constructed with square brackets, separating items with commas: [a,b.c])
--> tuple (used to store data, constructed with round brackets, unidexed and unmalleable; does allow duplicates)
Mapping Type:
--> dictionaries (used to store data values in value pairs, E.g. this dict = {“brand”: “Ford”, “model”: Mustang})
Set Type:
--> set (used to store data in and unordered, not repeating and unindexed way meaning the order will change every time you print the set)
Boolean Types:
--> Boolean values (used to check if a expression is True or False, E.g. print(10>9))

Statements
- Statement: a unit of code that the Python interpreter can execute
- If statement: if the statement is true then the python id will execute the code
- Elif statement: if the pervious conditions were not true, then try this condition
- Else statement: if the previous conditions weren’t met then the id will execute the code from the else statement

Operators
- Fun fact: the + and * operators work for strings, Example one: print(“hello”+”cat”) >>> hellocat, Example two: print(3*”cat”) >>> catcatcat
- Remember that 1/2*pi will evaluate to pi/2
- Click here for extra information about python operators

Asking the User for input
- Sometimes it is very helpful to ask the user for input, E.g. x=input()

Comments
- Comments are used so other people reading your code can understand it better
- Single line comments start with a #, and Python will ignore them, E.g. #this is a comment
- Multiple line comments start and end with “”” and they are used to ignore multiple lines of code
